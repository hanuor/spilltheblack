Github as a server?

Being a lazy ass, I wanted to control the content of my personal website through my mobile and without interacting with the git terminal at all. Problem was, every time, I had to look up where my local repo was, had to open the terminal, put in some commands, manually change the html code, put in another set of commands and then at last the changes would go live.

A lot of effort would go into all of this. Hence, here is what I came up with.

If you would visit a Repo’s page on Github, you would find something called Raw shining brightly on the center-right of your screen.

For those of you who don’t know what it is — Raw, simply to put, is the unprocessed version of your file’s content.

So if you just click the Raw button, you would see that it returns a text to you.

Kind of how a REST Api works. Interesting.

We know for a fact that Github allows us to commit our code without the use of terminal.

So, the idea is to use Github as a server from where we can ping or request information every time we want it. And for that we can assume that the Raw format is actually an Api returning the requested information.

Warning — Implementation is for Github Pages only.

Now, I have set up this link, which would return me the footer content of my html website.

And inside my js file, all I did was fetch the div and load the data using the URL.

And that’s pretty much how this works. The downside? Your content is out there, visible to everyone. But that’s how open source works. Right?
